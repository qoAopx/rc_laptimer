# 📱 RC Car Lap Timer — Web Client (BLE Receiver)

ESP32 ラップタイマーからの BLE 通知を受信し、ラップタイムの記録・表示・撮影を行うスマートフォン向け Web アプリです。
インストール不要でブラウザだけで動作します。

---

## 📷 システム概要

```text
[ESP32 ラップタイマー] ──BLE Notify──→ [スマートフォンブラウザ]
                                              │
                                    ┌─────────┴──────────┐
                                    │  Web Bluetooth API  │
                                    │  MediaDevices API   │
                                    │  Clipboard API      │
                                    └─────────────────────┘
```

ラジコンがセンサを通過するたびに ESP32 から BLE 通知が届き、ブラウザ上でリアルタイムにラップタイムを記録します。
カメラと連動して通過瞬間を自動撮影する機能も搭載しています。

---

## ✨ 機能

- **BLE リアルタイム受信** — Web Bluetooth API で ESP32 からラップタイムを直接受信
- **ベストラップ表示** — セッション中のベストタイムを自動判定・ハイライト
- **統計情報** — 総ラップ数・平均ラップタイムをリアルタイム集計
- **ラップ履歴テーブル** — 全ラップを新しい順に一覧表示
- **自動撮影** — ラップ確定時にカメラで自動撮影し、ラップタイムを画像に印字してダウンロード
- **写真履歴** — 撮影画像を画面内にグリッド表示
- **クリップボードコピー** — 全ラップタイムをテキスト形式でコピー
- **データクリア** — 計測データと写真履歴を一括リセット

## 📡 BLE 仕様

| 項目 | 値 |
| --- | --- |
| Service UUID | `12345678-1234-1234-1234-1234567890ab` |
| Characteristic UUID | `abcdefab-1234-5678-1234-abcdefabcdef` |
| 受信フォーマット | `Lap: 12.34`（秒単位） |
| 通信方式 | GATT Notify |

受信データから `Lap:\s*([\d.]+)` の正規表現で数値を抽出し、`MM:SS.cs` 形式に変換して表示します。

## 🚀 使い方

### 1. 接続

1. ESP32 ラップタイマーの電源を入れます
2. スマートフォンのブラウザで `index.html` を開きます
3. **CONNECT** ボタンをタップし、デバイス一覧から `RC_LAPTIMER` を選択します
4. フッターに `CONNECTED` と表示されれば接続完了です

### 2. 計測

- ラジコンがセンサを通過すると自動でラップタイムが画面に追加されます
- **最初の通過はスタート検出**として扱われ、ラップカウントには含まれません
- ベストラップが更新されると BEST LAP 欄が自動で書き換わります

### 3. 自動撮影

1. **START CAMERA** ボタンをタップしてカメラを起動します（背面カメラを使用）
2. カメラ映像がプレビューに表示されたらラジコンをコース上に向けます
3. ラップ確定のたびに自動撮影され、ラップ番号とタイムが印字された画像が端末に保存されます
4. 撮影済み画像は画面内の写真履歴にも表示されます

### 4. データ管理

| ボタン | 操作 |
| --- | --- |
| **COPY** | 全ラップタイムをクリップボードにコピー |
| **CLEAR** | 計測データと写真履歴をリセット |
| **DISCONNECT** | BLE 接続を切断 |

**COPY** で出力されるテキスト形式：

```text
Lap 1: 00:12.34
Lap 2: 00:13.01
Lap 3: 00:11.89
```

## 🌐 動作環境

| 項目 | 要件 |
| --- | --- |
| ブラウザ | Chrome / Edge（Android・PC） |
| プロトコル | HTTPS または localhost（Web Bluetooth の要件） |
| Bluetooth | BLE 対応デバイス |
| カメラ | 背面カメラ搭載デバイス推奨 |

> **⚠️ 注意** — Web Bluetooth API は **Safari / Firefox では未対応**です。
> iOS の場合は **Bluefy** などの Web Bluetooth 対応ブラウザが必要です。
> HTTP 環境では Bluetooth・カメラともに動作しません。必ず HTTPS か localhost で配信してください。

---

## ⚙️ 技術的なポイント

- **Web Bluetooth API** — ネイティブアプリなしで BLE デバイスと直接通信
- **GATT Notify 購読** — `startNotifications()` でラップ受信をイベント駆動で処理
- **MediaDevices API** — `facingMode: "environment"` で背面カメラを優先起動
- **Canvas API** — 撮影フレームにラップタイムをテキスト印字して PNG 出力
- **Clipboard API + フォールバック** — 非 SSL 環境や旧ブラウザでも `execCommand` でコピー対応
- **スタート通過を除外した集計** — 初回通過（計測開始）をラップカウントから除外して正確な統計を算出
